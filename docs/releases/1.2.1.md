# DEGIRO Importer 1.2.1

## Highlights

- **Manifest-declared navigation.** Sidebar navigation now comes from
  `contributes.links.sidebar` in the manifest instead of a runtime
  `ctx.sidebar.addItem` call. The runtime registers only the route renderer,
  with an id (`main`) that exactly matches the manifest's declared route id.
  This removes duplicate-configuration ambiguity and lets the host know about
  the page before the addon boots.
- **Host-dependency single source of truth.** The Vite external list, the
  manifest `hostDependencies`, and the `package.json` peer dependencies now all
  derive from the SDK's exported `HOST_DEPENDENCIES` map, eliminating drift
  between four previously hand-maintained copies.
- **Sandbox contract guards.** A source-level scan now rejects
  `localStorage`/`sessionStorage`/`indexedDB`/`XMLHttpRequest`/`WebSocket`/
  `EventSource`/`fetch()` in the addon's own source, preventing accidental use
  of APIs that throw or are blocked in the opaque-origin sandbox.
- **Tighter SDK pin.** The build SDK dev dependency is pinned to `~3.6.1`
  (admits the 3.6.2 patch when published, forbids a 3.7.0 that could compile
  against APIs unavailable on a 3.6 host). The manifest host range stays
  `^3.6.0`.
- Removed `sidebar.addItem` from the requested `ui` permissions (navigation is
  manifest-declared).

## Compatibility

- Wealthfolio: >= 3.6.1
- Add-on SDK: 3.6.x (built against ~3.6.1)
- Tested with Wealthfolio: 3.6.1 (host integration testing is a separate
  release gate)
- DEGIRO Account statement CSV exports in Dutch or English header variants

## Notes

- No change to import parsing semantics. The parser, mapping, reconciliation,
  validation, and duplicate-detection logic are unchanged.
- This release targets the published `@wealthfolio/addon-sdk@3.6.1`, which
  supports only the `render`-callback route model. The host-managed `component`
  route model is unreleased (lands in 3.6.2); migrating to it is future work,
  not a defect of this release. The `onRendered` host acknowledgement callback
  is retained defensively (it is undocumented in the SDK type but present in
  the 3.6.1 runtime context).

See the [installation guide](../INSTALL.md) and [input format](../FORMAT.md)
before importing.
